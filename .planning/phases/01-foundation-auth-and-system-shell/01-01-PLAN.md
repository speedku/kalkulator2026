       - NO imports of Prisma or bcryptjs (these crash Edge runtime)
       - Export `authConfig` satisfies NextAuthConfig
       - `pages.signIn: '/login'`, `pages.error: '/login'`
       - JWT callback: attach user.id and user.role to token
       - Session callback: attach token.id and token.role to session.user
       - Authorized callback: redirect unauthenticated to /login, redirect authenticated away from auth pages
       - `session: { strategy: 'jwt', maxAge: 7 * 24 * 60 * 60 }` (7-day session, AUTH-02/AUTH-04)
       - `providers: []` (populated in auth.ts)

    2. Create `src/auth.ts` (full Node.js config) exactly as shown in 01-RESEARCH.md Pattern 1:
       - Import and spread authConfig
       - Credentials provider with authorize function:
         a. Look up user by email (lowercase) with select for id, email, name, passwordHash, role, isActive, emailVerified, failedLoginAttempts, lockedUntil
         b. Check isActive and emailVerified
         c. AUTH-07: Check lockedUntil (if locked and not expired, return null)
         d. Verify password with bcryptjs.compare
         e. On failure: increment failedLoginAttempts, lock if >= 5 (15 min lockout)
         f. On success: reset failedLoginAttempts to 0, clear lockedUntil, update lastLogin
         g. Return user object { id, email, name, role }
       - Export auth, signIn, signOut, handlers

    3. Create `src/app/api/auth/[...nextauth]/route.ts`:
       - Import handlers from `@/auth`
       - Export GET and POST handlers
       - This is the Auth.js catch-all route

    4. Create `src/middleware.ts` exactly as shown in 01-RESEARCH.md Pattern 1:
       - Import NextAuth from 'next-auth' and authConfig from './auth.config' (NOT from './auth')
       - Export default NextAuth(authConfig).auth
       - Matcher: exclude api/auth, _next/static, _next/image, favicon.ico, and public auth pages

    5. Create `src/lib/dal/auth.ts` (DAL auth module):
       - `import 'server-only'` at top (prevents client import)
       - `getCurrentUser()` wrapped with `cache()` from React:
         a. Get session via auth()
         b. Look up user by session.user.id
         c. Return user or null (check isActive)
       - `hasPageAccess(pageId: string)`:
         a. Get current user
         b. Admin role always returns true
         c. Query userPermission for userId + pageId
         d. Return canAccess boolean
       - `requireAuth()`: calls getCurrentUser, throws redirect to /login if null
       - `requireAdmin()`: calls getCurrentUser, throws redirect or 403 if not admin
