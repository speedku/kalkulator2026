  <action>
    1. Create `src/lib/validations/users.ts`:
       - `createUserSchema`: name (min 2), email (email format), password (min 8), role (enum: admin/user)
       - `updateUserSchema`: name, email, role (all optional), isActive (boolean optional)

    2. Create `src/lib/validations/permissions.ts`:
       - `updatePermissionsSchema`: userId (number), permissions (array of { pageId: string, canAccess: boolean, canSee: boolean })

    3. Create `src/lib/validations/access-codes.ts`:
       - `createAccessCodeSchema`: code (min 3, uppercase transform), description (optional), maxUses (min 1, default 1), expiresAt (optional date)
       - `updateAccessCodeSchema`: isActive (boolean), description, maxUses, expiresAt

    4. Create `src/lib/dal/users.ts`:
       - `import 'server-only'`
       - All functions call `requireAdmin()` first (from dal/auth.ts)
       - `getUsers(params: { search?, page?, perPage?, role? })`: paginated user list with search on name/email, returns { users, total, page, perPage }
       - `getUserById(id: number)`: single user with permissions and recent activity
       - `createUser(data)`: create user with hashed password (bcryptjs cost 12), log activity
       - `updateUser(id, data)`: update user fields, log activity
       - `toggleUserActive(id)`: toggle isActive, log activity
       - `deleteUser(id)`: soft delete by setting isActive: false (never hard delete — FK constraints). Log activity.

    5. Create `src/lib/dal/permissions.ts`:
       - `import 'server-only'`
       - `requireAdmin()` guard on all functions
       - `getUserPermissions(userId: number)`: all permissions for a user with page metadata
       - `getAvailablePages()`: return list of all valid page_ids from the navigation config
       - `updateUserPermissions(userId, permissions[])`:
         a. For each permission: upsert UserPermission record
         b. For each change: create UserPermissionsHistory entry (action: 'granted' or 'revoked', changed_by current admin, IP, user agent)
         c. Log activity for each change
       - `bulkSetPermissions(userId, pageIds[], canAccess, canSee)`: batch toggle

    6. Create `src/lib/dal/access-codes.ts`:
       - `import 'server-only'`
       - `requireAdmin()` guard
       - `getAccessCodes(params: { page?, perPage?, search? })`: paginated list with usage stats
       - `getAccessCodeById(id)`: single code with users who used it
       - `createAccessCode(data)`: create with createdBy = current admin id, log activity
       - `updateAccessCode(id, data)`: update fields, log activity
       - `deleteAccessCode(id)`: set isActive: false, log activity

    7. Create `src/lib/dal/notepad.ts`:
       - `import 'server-only'`
       - `requireAuth()` guard (NOT requireAdmin — all users can use notepad)
       - `getNote()`: get current user's note (UserNote with userId = getCurrentUser().id). Return null if no note yet.
       - `saveNote(content: string, contentPlain: string)`:
         a. Get current user
         b. Upsert UserNote (create if first save, update if exists)
         c. Extract hashtags from content (regex for #word patterns)
         d. Calculate wordCount and characterCount from contentPlain
         e. Create UserNoteVersion with incremented versionNumber
         f. Upsert UserNoteTag entries for extracted tags (increment usageCount)
         g. Log activity
       - `getNoteVersions(page?, perPage?)`: paginated version history for current user's note

    8. Create Server Actions in `src/lib/actions/`:
       - `users.ts`: createUserAction, updateUserAction, toggleUserActiveAction — each validates with Zod, delegates to DAL, calls revalidatePath
       - `permissions.ts`: updatePermissionsAction — validates, delegates to DAL, revalidatePath
       - `access-codes.ts`: createAccessCodeAction, updateAccessCodeAction, deleteAccessCodeAction — validates, delegates, revalidatePath
       - `notepad.ts`: saveNoteAction — validates content not empty, delegates to DAL, revalidatePath

    All Server Actions follow the pattern: validate with Zod -> call DAL function -> revalidatePath -> return { success } or { error }.
    All DAL functions log activity via logActivity() from dal/activity-log.ts.
    All admin DAL functions guard with requireAdmin().
    Notepad DAL functions guard with requireAuth() (any authenticated user).
