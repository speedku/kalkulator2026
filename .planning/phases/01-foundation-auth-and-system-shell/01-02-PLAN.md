    6. Create `src/lib/email.ts` — Email utility:
       - Configure Nodemailer transport with SMTP_HOST, SMTP_PORT, SMTP_USER, SMTP_PASS from env
       - `sendPasswordResetEmail(to: string, token: string)`: sends email with reset link (`${process.env.AUTH_URL}/reset-password?token=${token}`)

       - Use `SMTP_FROM` env var as sender

    7. Create `src/lib/actions/auth.ts` — Server Actions:
       a. `loginAction(prevState, formData)`:
          - Validate with loginSchema (Zod)
          - Call signIn('credentials', { email, password, redirect: false })
          - Log activity (activityType: 'auth', action: 'login')
          - Return { success: true } or { error: string }
          - Handle CredentialsSignin error for wrong credentials
          - Handle lockout error message

       b. `registerAction(prevState, formData)`:
          - Validate with registerSchema
          - Verify access code exists, is active, not expired, not at max uses
          - Check email not already taken
          - Hash password with bcryptjs (cost 12, matching PHP hashes)
          - Create user (isActive: true, emailVerified: false)
          - Generate verification token, store hash in DB
          - Send verification email
          - Increment access code currentUses
          - Log activity (activityType: 'auth', action: 'create')
          - Return success message about checking email

       c. `forgotPasswordAction(prevState, formData)`:
          - Validate with forgotPasswordSchema
          - Look up user by email (don't reveal if user exists — always say "check email")
          - Generate crypto.randomUUID() token
          - Hash token with crypto.createHash('sha256'), store in resetToken + resetTokenExpires (1 hour)
          - Send password reset email
          - Log activity
          - Return success message

       d. `resetPasswordAction(prevState, formData)`:
          - Validate with resetPasswordSchema
          - Hash the URL token with sha256, find user with matching resetToken where resetTokenExpires > now
          - Hash new password with bcryptjs (cost 12)
          - Update user password, clear resetToken/resetTokenExpires
          - Log activity
          - Return success with redirect to login

    8. Create `src/app/(auth)/login/page.tsx`:

    10. Create `src/app/(auth)/forgot-password/page.tsx`:

    11. Create `src/app/(auth)/reset-password/page.tsx`:

        - Hidden input for token
        - Success redirects to login

    For ALL form pages: Use `useActionState` (React 19) NOT the old `useFormState`. Place the submit button inside the form as a child component that uses `useFormStatus` for pending state.
